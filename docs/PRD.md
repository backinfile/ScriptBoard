# ScriptBoard MVP 产品需求文档

**状态**：需求冻结  
**冻结日期**：2026-07-20  
**适用版本**：MVP

## 1. 产品定义

ScriptBoard 是一个自托管、单机、单管理员的受管目录文件服务器，并为其中的可信脚本提供本地执行、实时日志、历史追踪、快捷执行、内置计划和可选本地 Git 版本保护。

它不是多用户运维平台，也不隔离恶意脚本。脚本直接继承 ScriptBoard 服务进程的操作系统身份与环境；默认服务安装使用 Linux root 或 Windows LocalSystem，因此管理员会话、受管文件和脚本均属于宿主机完全可信边界。

## 2. 产品目标

管理员能够完成以下闭环：

```text
登录 → 管理受管目录 → 选择脚本与参数 → 启动执行
     → 实时查看输出 → 停止执行 → 查询不可删除的历史
     → 保存快捷执行或计划 → 必要时从本地 Git 恢复误改文件
```

MVP 优先保持单服务进程、SQLite、单机文件系统与平台原生服务部署，不引入运行时集群依赖。

## 3. 信任与权限边界

- 只有一个 admin 账号，没有角色、用户组或逐脚本授权。
- 管理员上传和启动的脚本视为可信代码，不提供沙箱。
- ScriptBoard 不在应用内部提权、降权或切换用户；脚本继承服务进程身份与环境。
- Windows 服务默认使用 LocalSystem；Linux systemd 服务默认使用 root。安装者可在操作系统服务配置中改用其他账号。
- 明文 HTTP 只能监听回环地址。非回环访问必须使用内置 TLS，或由同机可信 HTTPS 反向代理转发到回环后端。
- 所有文件、日志、SSE、变量和执行入口必须验证 admin Session；匿名仅允许登录入口和无敏感内容的静态资源。

## 4. 信息架构

主导航：

```text
文件 | 执行记录 | 快捷执行 | 计划 | 变量 | 审计
```

右侧显示当前运行数、admin 菜单和退出登录。回收站从文件页面进入，不占主导航。

主要路由：

- `/login`
- `/files/*`
- `/runs`
- `/runs/{id}`
- `/quick-runs`
- `/schedules`
- `/variables`
- `/audit`
- `/settings/account`
- `/trash`

## 5. 认证与会话

### 5.1 管理员初始化

- 首次启动自动创建用户名 `admin`。
- 自动生成至少 128 bit 随机熵的初始密码，写入权限受限的一次性文件。
- 首次登录强制修改密码；成功后删除一次性文件。
- Windows 托盘可复制初始密码。
- 启动时可用用户名、密码文件或明文密码覆盖数据库凭据。覆盖值不同则重设账号并撤销全部 Session。
- 网页修改凭据时，如果启动配置仍提供覆盖值，必须提示下次重启会再次覆盖。

### 5.2 密码

- 只使用带版本化参数的 Argon2id，每个密码使用独立随机 Salt。
- 最少 12 个 Unicode 字符，最多 256 个 UTF-8 字节。
- 不强制字符类别组合，允许空格、中文和密码短语；不能与用户名相同。
- 修改密码要求当前密码；本机最高权限 CLI 可重设。
- 登录成功时可自动升级过旧的 Argon2id 参数。

### 5.3 Session

- 允许多个浏览器同时登录。
- 空闲 12 小时失效，绝对有效期 7 天。
- 浏览器只保存随机 Session ID，服务端只保存其哈希。
- 修改用户名或密码撤销全部 Session；普通退出只撤销当前 Session，并提供退出所有设备。
- 服务重启不使有效 Session 自动失效。
- Cookie 设置 HttpOnly、SameSite；HTTPS 模式强制 Secure。
- 所有状态修改请求必须使用 POST 等非 GET 方法并验证 CSRF Token。

### 5.4 登录限速

- 连续失败 5 次后开始指数退避，从 2 秒增加到最长 5 分钟。
- 同时按来源 IP 和唯一 admin 账号统计。
- 不永久锁定账号；本机 CLI 可清除限速状态。
- 登录成功、失败和限速均写审计事件，不记录密码。

## 6. 文件管理

### 6.1 目录边界

- 受管根目录与内部状态目录必须为互不包含的真实路径。
- Linux 默认：`/var/lib/scriptboard/managed` 与 `/var/lib/scriptboard/state`。
- Windows 默认：`%ProgramData%\ScriptBoard\managed` 与 `%ProgramData%\ScriptBoard\state`。
- 受管根目录内所有普通文件和目录可见，保留目录除外。
- 符号链接、Junction、Reparse Point、跨设备挂载和卷挂载显示为受限条目，但不能进入、读取、下载、覆盖或执行。
- 不跨越受管根目录所在文件系统或卷。
- 硬链接按普通文件显示；重叠执行和租约按操作系统文件 ID 识别。

### 6.2 列表与搜索

- 每页 100 个条目，目录优先，默认自然名称排序。
- 支持名称、类型、大小和修改时间排序。
- 搜索只匹配当前目录，不递归索引内容。
- 单目录扫描超过 100,000 个条目时中止并提示拆分目录。
- 显示文件类型、大小、修改时间、执行状态和 Git 保护状态。

### 6.3 上传与创建

- 支持一次上传多个普通文件，以及网页新建目录。
- 不支持文件夹上传、服务端解压、符号链接或特殊文件上传。
- 默认限制：单文件 1 GiB、单请求 2 GiB、单批 100 个；可配置但不能无限。
- 上传全程流式写入同目录随机临时文件，不把完整内容载入内存。
- 文件名拒绝路径分隔符、NUL、`.` 和 `..`，其余 Unicode 尽量保留。
- 同名默认拒绝；明确二次确认后才原子替换，旧版本进入回收站。
- 目录、链接、特殊文件及运行中脚本不能被上传替换。
- Linux 新文件默认请求 0644、目录 0755，并受 umask 限制；Windows 继承目标目录 ACL。

### 6.4 预览、编辑与下载

- UTF-8、无 NUL、最大 1 MiB 的普通文件支持预览和编辑。
- PNG、JPEG、GIF、WebP 支持受限内嵌预览；扩展名与内容不一致时不预览。
- SVG、HTML、PDF、Office、音视频和未知格式只下载。
- 文本保存使用内容摘要做乐观并发检查，并通过同目录临时文件原子替换；旧版本进入回收站。
- 运行中脚本不可编辑。
- 只支持单个普通文件流式下载，并支持 HTTP Range；不下载目录或动态生成 ZIP。
- 下载使用 `Content-Disposition: attachment` 和 `X-Content-Type-Options: nosniff`。

### 6.5 重命名、移动与引用

- 网页重命名或移动脚本/目录时，同步更新受影响的快捷执行项与计划项路径。
- 文件系统变更成功、数据库更新失败时尽力回滚；无法回滚则报告不一致。
- 绕过网页的外部移动不猜测身份，引用显示失效。
- 历史 Run 永远保留执行当时路径。

### 6.6 回收站

- 网页删除将内容移动到受管根目录保留区域 `.scriptboard-trash/`。
- 普通文件列表隐藏回收站，专用页面支持恢复与永久清理。
- 恢复保留原路径，遇到路径冲突时拒绝覆盖。
- 默认不自动清理回收站。
- 删除被计划或快捷项引用的脚本时，先列出影响并二次确认；计划自动禁用，快捷项失效。
- 恢复原路径后快捷项可恢复可用，计划不会自动重新启用。

### 6.7 Linux 可执行位

- Linux 普通文件只提供所有者执行位 `u+x` / `u-x` 开关。
- 不提供任意 chmod、chown、chgrp 或目录权限编辑。
- Windows 隐藏该功能。

## 7. 本地 Git 版本保护

### 7.1 定位与启用

- Git 保护默认关闭，由管理员在网页启用或停用。
- 它只用于恢复误改，不是备份，也不进行远程同步。
- 使用受管根目录中的单一本地仓库，专用分支为 `scriptboard-managed`。
- 默认自动跟踪不超过 10 MiB 的普通文件；阈值可配置但不能无限。
- `.git/`、回收站、上传临时文件、受限链接和挂载强制排除。
- 遵循 `.gitignore`；被忽略文件不强制跟踪。
- 依赖系统 Git CLI，可配置绝对路径；功能关闭时不要求 Git 已安装。

### 7.2 接管外部仓库

- 允许接管工作树根恰好等于受管根目录的普通非 bare 仓库。
- `.git` 必须为真实本地目录，不能是链接、挂载或 gitfile。
- 接管前展示当前分支、提交、远程与脏状态，并执行完整性检查。
- 保留对象、提交、分支和标签；备份后替换为最小安全仓库配置。
- 禁用 Hooks、外部 diff、clean/smudge filter、fsmonitor、credential helper、子模块和所有远程操作。
- 不支持 Git LFS 或可执行 `.gitattributes`；发现时拒绝接管或进入异常状态。
- 不配置全局 `safe.directory=*`；所有权不一致必须明确确认。

### 7.3 提交模型

- 固定提交身份：`ScriptBoard <scriptboard@localhost>`。
- 提交信息由应用生成，并带操作类型、Run ID 和审计事件 ID 等 trailers。
- 无活动 Run 时，网页文件修改完成后立即提交。
- 第一个 Run 启动前创建 pre-run checkpoint；任何 Run 活动期间不执行 Git add/commit；最后一个 Run 结束后创建 post-run checkpoint。
- 并发批次提交只表示全局文件时间线，不声称精确归因到单个 Run。
- 不后台轮询，也不使用文件系统 watcher；外部变化只在启停、网页操作、Run 批次边界或手动检查点时进入历史。
- 异常退出后保留基线与脏工作树，重启不自动提交。

### 7.4 启停与异常

- 只有无活动 Run 时才能切换 Git 保护。
- 首次启用、接管、停用和重新启用分别创建 baseline、adoption、final 和 re-enable checkpoint。
- 停用不删除 `.git/` 或历史。
- pre-run checkpoint 失败时拒绝启动脚本。
- 受保护文件的网页变更提交失败时尽力回滚。
- post-run checkpoint 失败使 Git 状态异常；异常状态禁止新执行和受保护文件修改，直到重试成功或管理员明确停用。
- 因过大或 `.gitignore` 而未跟踪的脚本仍可从手动、快捷和计划入口执行，只显示未保护原因。

### 7.5 大小、维护与恢复

- 已跟踪文件超过阈值时从当前索引移除但保留工作树和历史；缩小后自动重新加入。
- `.git/` 默认上限 5 GiB，80% 时警告；达到上限后停止新提交，不自动删除可达历史。
- 只在无活动 Run 时运行禁用 Hooks 的安全 `git gc`。
- 文件详情支持查看单文件历史；文本可看 diff，二进制显示大小与摘要。
- 只支持将单个文件恢复到历史版本，恢复产生新提交，不改写历史。
- 不提供整仓/目录恢复、branch、merge、rebase、reset、cherry-pick、fetch、pull、push、clone 或 submodule。

## 8. 脚本识别与执行器

### 8.1 无注册执行

- 受支持扩展名的普通文件无需注册，直接在原位置执行。
- 不复制脚本快照；Run 保存启动时路径、SHA-256 和 Git 保护状态。
- 工作目录固定为脚本父目录。
- 脚本继承服务进程环境；额外注入 `SCRIPTBOARD_RUN_ID` 和 `SCRIPTBOARD_SCRIPT_PATH` 等非敏感元数据。

### 8.2 默认执行器链

Linux：

```text
.sh  → bash → sh
.py  → python3 → python
.ps1 → pwsh
```

Windows：

```text
.ps1 → pwsh.exe → powershell.exe
.py  → py.exe -3 → python.exe
.bat → cmd.exe
.cmd → cmd.exe
.sh  → bash.exe
```

- 候选可包含固定前缀参数，可按 PATH 查找或配置绝对路径。
- 只有执行器不存在或进程无法创建时才回退；脚本成功启动后，非零退出、超时或取消均不换执行器重跑。
- Windows Terminal 不是执行器。托盘可在检测到 `wt.exe` 时提供“在受管目录打开 Terminal”。
- WSL 不作为默认 `.sh` 执行器。

## 9. 参数与变量

- 参数输入采用跨平台简化 shellwords：空白分隔、单双引号和反斜杠转义。
- 不执行变量展开以外的环境展开、通配符、管道、重定向、`&&`、`;` 或命令替换。
- 不提供 Shell 模式。
- 变量名格式为 `[A-Z][A-Z0-9_]{0,63}`，最多 1,000 个。
- 变量均为普通明文值，最大 4 KiB，可为空；不提供加密、遮罩或秘密类型。
- 引用格式为 `{{NAME}}`，必须独占一个参数，不能嵌入文本。
- 参数模板最大 16 KiB，最多解析为 256 个参数；解析后按实际执行器限制再次校验。
- 被快捷项或计划引用的变量不能删除；重命名时自动更新活动引用。
- Run 同时保存原始模板、模板参数数组和启动时解析后的实际参数数组。
- 大块输入通过受管文件路径传递，不存在“运行附件”概念。

## 10. 执行生命周期

### 10.1 并发与重叠

- 不设置全局并发上限，不建立队列。
- 手动或快捷执行同一脚本已有活动实例时，显示活动数量和启动时间并二次确认。
- 计划默认允许同一脚本重叠；计划项可选择禁止重叠，被跳过时记录计划触发事件而不创建 Run。
- 重叠和租约按规范化真实路径及操作系统文件 ID 判断。

### 10.2 执行租约

- 从启动请求被接受到该文件所有活动实例结束，禁止通过网页覆盖、编辑、移动、重命名或删除脚本。
- 脚本所有祖先目录禁止移动、重命名和删除，避免绕过租约。
- 同目录其他文件不锁定，系统也不承诺阻止绕过网页的磁盘修改。

### 10.3 超时

- 默认无超时。
- 手动执行可指定本次超时；快捷项保存来源 Run 超时；计划保存自身超时。
- 可配置允许的最大超时，默认 24 小时。
- 达到超时时先进入 `timing_out` 并请求优雅终止，默认等待 30 秒后自动强杀完整进程树。

### 10.4 手动停止

- 第一次点击停止：请求优雅终止，状态变为 `stopping`，按钮变为“强制停止”。
- 第二次点击：立即强杀完整进程树。
- 第一次手动停止后不按固定时间自动强杀。

### 10.5 服务生命周期

- 正常停服先停止接受新执行和计划触发，再优雅终止全部活动进程树；等待 30 秒后强杀残留，Run 记为 `cancelled`，原因是服务停止。
- Linux 使用独立进程组和父进程死亡信号；Windows 使用启用关闭即终止的 Job Object。
- 异常服务退出后，尽量由操作系统连带终止脚本；下次启动将原活动 Run 记为 `disconnected`。
- 浏览器刷新可通过 Run ID 与事件序号重新连接 SSE；服务重启后只允许读取已有日志，不重新接管进程。

## 11. Run 状态机

```text
starting → running → succeeded
                  → failed
                  → stopping → cancelled
                  → timing_out → timed_out
                  → disconnected
starting → failed
```

- 启动前校验失败不创建 Run；进程创建阶段失败创建 `failed` Run。
- 正常退出码 0 为 `succeeded`，非零为 `failed`。
- 停止或超时一旦触发，最终状态由该原因决定，不再被随后退出码改写。
- 不存在 `queued`。
- Run 元数据不可由管理员删除。

## 12. 日志与历史

### 12.1 执行日志

- 每个 Run 使用一份追加式事件日志，事件包含稳定序号、时间、stdout/stderr 来源和原始字节。
- 顺序表示 ScriptBoard 从两条管道读取到的顺序，不承诺进程内部精确时序。
- 页面不模拟终端：不分配 PTY、不接受 stdin、不执行 ANSI 控制序列。
- 保留换行、回车和 Tab，其他控制字符转义；无效 UTF-8 使用替换字符并标记。
- 每个 Run 默认最多 100 MiB：固定保留最初 5 MiB，并滚动保存最新约 95 MiB，中间插入截断事件。
- 即使日志被丢弃或写入失败，也持续排空 stdout/stderr，避免进程阻塞。

### 12.2 SSE 与页面容量

- SSE 使用事件序号与 `Last-Event-ID` 重连。
- 页面初始最多加载最近 5 MiB 或 5,000 条事件，取较小者。
- 浏览器最多保留 10,000 条事件，超出后只从 DOM 移除。
- 支持暂停显示、自动跟随与用户滚动后暂停跟随。

### 12.3 保留

- Run 元数据永久且不可手动删除。
- 日志默认保留 90 天，全局总量上限 1 GiB；先清理最旧已结束 Run 日志并标记 `log_expired`。
- 活动与 `disconnected` Run 不参与自动日志清理。
- 触发成功关系随 Run 永久保留；未创建 Run 的计划触发明细保留一年，之后按计划、日期和结果聚合为永久计数。

## 13. 快捷执行

- 管理员从历史 Run 主动保存具名快捷执行项，不自动为所有历史生成入口。
- 保存脚本相对路径、参数模板/数组、超时、来源 Run ID、名称和排序。
- 默认单击即通过带 CSRF Token 的 POST 启动并跳转实时日志。
- 同脚本已有活动实例时改为二次确认；可将个别快捷项配置为始终确认。
- 文件失效时快捷项不可用；网页移动会更新路径；Run 来源删除不存在，因为 Run 不可删除。

## 14. 内置计划

- 不读取或修改系统 crontab。
- 计划保存在 SQLite，由服务进程内调度器执行，Windows/Linux 行为一致。
- 采用五段 cron（分钟、小时、日、月、星期），最小粒度一分钟；不支持秒或 `@every`。
- 页面提供常用预设、高级表达式和未来五次触发预览。
- 所有计划使用实例统一时区，默认宿主时区，可用 IANA 名称覆盖；数据库时间戳保存 UTC。
- 不对夏令时跳变实现特殊补偿或去重。
- 服务停机期间不补执行；重启后只从下一次未来时间继续，并汇总记录错过次数。
- 计划保存脚本路径、参数模板、超时、启用状态和重叠策略。
- 指向被删除脚本的计划自动禁用且不会随恢复自动启用。

## 15. 审计

- 审计范围：登录、退出、文件变更、回收站、执行启动与停止、计划、变量、快捷项、账号/配置修改、Git 启停与恢复、日志清理等。
- 记录时间、操作、目标、结果和请求来源地址。
- 不记录密码、Cookie、CSRF Token、变量值、文件内容或请求正文。
- 默认保留一年，可下载，不允许网页逐条修改。
- 审计日志、Run 日志和服务诊断日志相互独立。

## 16. 服务、托盘与网络

### 16.1 CLI

```text
scriptboard serve
scriptboard service install|uninstall|start|stop|restart|status
scriptboard admin reset
scriptboard config validate
scriptboard doctor
scriptboard version
```

- Windows 直接使用服务控制管理器，不依赖 NSSM。
- Linux 只正式支持 systemd。
- `service uninstall` 不删除配置、受管文件、状态或 Git 历史。
- 不提供用户备份/恢复命令。

### 16.2 Windows 托盘

- 服务默认随系统启动；托盘默认随当前用户登录启动，并禁止重复实例。
- 无主窗口。菜单包含状态、打开网页、启动、停止、重启、打开受管目录、查看服务日志、在受管目录打开 Terminal（可用时）、登录自启动和退出。
- 启停转换期间禁用重复操作；停止/重启有活动 Run 时显示数量并确认。
- 退出托盘不停止服务。
- 就绪状态同时检查 Windows 服务状态与 `/login` 可访问性；进程存在但 HTTP 未就绪显示“启动中”。

### 16.3 网络

- 默认 `http://localhost:8787`，监听 `127.0.0.1` 与 `::1`。
- 端口可配置；冲突时拒绝启动，不随机换端口。
- 非回环监听必须配置 TLS；不内置 ACME。
- 可信反向代理必须显式配置，才接受转发的客户端地址。
- 不提供公共 API、API Token、Basic Auth 或远程 CLI。

## 17. 配置、状态与诊断

### 17.1 配置优先级

```text
内置默认值 < YAML 配置文件 < SCRIPTBOARD_ 环境变量 < 命令行参数
```

- Linux 默认配置：`/etc/scriptboard/config.yaml`。
- Windows 默认配置：`%ProgramData%\ScriptBoard\config.yaml`。
- `--config` 可指定其他文件。
- 网页只读展示最终生效配置与非敏感来源，MVP 不在线修改启动配置。

### 17.2 SQLite

- 使用纯 Go SQLite 驱动。
- 启用 WAL、`synchronous=FULL`、foreign keys 和 busy timeout。
- 高频 Run 输出不逐事件写数据库。
- 数据库 schema 只向前迁移；迁移前创建内部安全快照，失败事务回滚并拒绝启动。
- 检测到数据库完整性异常时拒绝提供执行能力，只允许本机诊断和人工恢复流程。

### 17.3 磁盘与日志

- 状态卷不足 100 MiB 时拒绝新执行。
- 受管卷不足 100 MiB 时拒绝上传和文本保存。
- 已运行脚本不因低磁盘自动终止；日志写入失败标记 `log_incomplete`。
- 服务诊断日志位于状态目录，单文件 10 MiB，保留 5 个；Linux 同时输出到 stderr/journald。
- 服务日志不得输出密码、Session、变量值、完整环境或请求正文。

### 17.4 doctor

`scriptboard doctor` 只读检查配置、目录边界与权限、SQLite、Run/日志对应、执行器、Git、端口、TLS、服务注册和磁盘空间。默认不修复，也不输出认证信息或变量值。

## 18. 技术与发布

- Go 单模块。
- 标准库 `net/http`、`html/template`、少量原生 JavaScript、SSE、纯 Go SQLite、`embed` 资源。
- 不引入 Node.js 构建链、SPA、Redis 或消息队列。
- Windows 托盘为同模块独立 GUI 可执行文件。
- 正式平台：Windows 10/11、Windows Server 2019+、systemd Linux；amd64 与 arm64。
- Windows 发布 ZIP，Linux 发布 tar.gz，并提供 SHA-256；MVP 不制作 MSI/DEB/RPM。
- 不自动更新，不联网检查版本。
- MVP 界面、错误、托盘和 CLI 帮助只提供简体中文；结构化标识保持英文。
- 支持当前及前一个主要版本的 Chrome、Edge、Firefox、Safari，并提供可用的移动端核心操作。
- 正式支持每台主机单实例；状态目录持有排他实例锁。

## 19. 非目标

- 多用户、角色、用户组、LDAP、OAuth、SSO
- 不可信代码沙箱或逐脚本身份切换
- 公共 API、API Token、远程 CLI
- DAG、流水线、多服务器和 Kubernetes 编排
- Docker 作为正式部署方式
- 系统 crontab 集成
- Git 远程同步、子模块、LFS 和外部过滤器
- 用户备份/恢复命令
- 邮件、聊天通知和插件系统
- 交互式终端、PTY 和 stdin
- 文件夹上传、ZIP 打包下载和服务端解压
- 通用 chmod/chown 文件权限管理
- 自动更新
- 多语言和正式多实例支持
