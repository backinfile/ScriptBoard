# ScriptBoard MVP 验收标准

本文是冻结需求的可验证交付标准。除明确标注平台特有的条目外，Windows 与 Linux 均需通过。

## 1. 认证与安全

- [ ] 首次启动自动创建 admin 和随机初始密码；首次登录强制修改。
- [ ] Argon2id 哈希包含版本化参数和独立 Salt，数据库不出现明文密码。
- [ ] 密码不满足 12 Unicode 字符、超过 256 UTF-8 字节或等于用户名时被拒绝。
- [ ] 未登录访问文件、下载、日志、SSE、变量、计划和执行路由均被拒绝。
- [ ] 所有状态修改路由拒绝缺失或错误 CSRF Token。
- [ ] Session 空闲 12 小时、绝对 7 天后失效；服务重启不使有效 Session 失效。
- [ ] 修改用户名/密码撤销全部 Session；退出只撤销当前 Session。
- [ ] 登录失败 5 次后出现指数退避，且不存在永久锁死。
- [ ] 明文 HTTP 配置非回环监听时服务拒绝启动。
- [ ] HTTPS 非回环模式设置 Secure Cookie；可信代理之外的转发 IP 头被忽略。
- [ ] 服务日志、Run 错误和审计中不出现密码、Cookie、CSRF、变量值或完整环境。

## 2. 文件边界

- [ ] 受管根目录与状态目录相同、互相包含或权限不足时拒绝启动。
- [ ] `..`、编码路径穿越、绝对路径注入和路径分隔符文件名均不能逃逸受管根目录。
- [ ] 符号链接、Junction、Reparse Point 和跨设备挂载可见但不可进入、读取、下载、覆盖或执行。
- [ ] `.git/`、回收站和上传临时文件不会出现在普通文件列表。
- [ ] 硬链接到同一文件时，共享重叠检测与执行租约。
- [ ] 单目录 100,001 个条目时请求被有界终止，而不是无限占用内存。

## 3. 上传、编辑、下载与回收站

- [ ] 多文件上传逐文件返回结果；文件夹、链接和特殊文件被拒绝。
- [ ] 单文件 1 GiB、单请求 2 GiB、每批 100 个默认上限生效，上传为流式写入。
- [ ] 同名上传默认拒绝；确认替换后旧文件进入回收站且新文件原子可见。
- [ ] 上传中断不会在目标路径留下半文件。
- [ ] UTF-8 无 NUL 且不超过 1 MiB 的文件可编辑；外部改动后保存发生冲突而不覆盖。
- [ ] 运行中脚本不能编辑、替换、移动或删除，其祖先目录不能移动或删除。
- [ ] 栅格图片按允许类型预览；SVG/HTML/PDF/Office/未知格式不内嵌。
- [ ] 单文件下载支持 Range；目录和多选下载不生成 ZIP。
- [ ] 删除进入回收站；冲突路径不能恢复；永久清理需要明确确认。
- [ ] 删除被引用脚本会禁用计划并使快捷项失效；恢复不自动重启计划。
- [ ] Linux 可切换 `u+x` 且不改变 group/other 位；Windows 不显示该操作。

## 4. Git 版本保护

- [ ] Git 未安装时核心应用可运行，启用 Git 时得到明确错误。
- [ ] 新仓库创建 `scriptboard-managed` 分支与 baseline 提交。
- [ ] 外部非 bare 仓库可在检查、净化、确认后接管；原分支和标签保留。
- [ ] `.git` 为链接、挂载、gitfile，或仓库含 LFS/可执行 filter 时拒绝接管。
- [ ] Git Hooks、外部 diff/filter、fsmonitor、credential helper、远程和子模块不会被执行。
- [ ] `.gitignore` 生效，应用保留路径始终强制排除。
- [ ] 10 MiB 内符合条件文件受保护，超过阈值显示未保护且仍可执行。
- [ ] 文件从小变大时从当前索引退出但历史保留；缩小后重新加入。
- [ ] 首个并发 Run 前产生 pre-run checkpoint，最后一个 Run 后产生单个 post-run checkpoint。
- [ ] 任意 Run 活动期间不能启停、恢复或运行 Git 维护。
- [ ] 无后台轮询；外部变化只在明确操作边界或手动检查点进入历史。
- [ ] pre-run 提交失败阻止执行；post-run 失败进入 abnormal 并阻止后续执行。
- [ ] 单文件可恢复到历史版本，并通过新提交完成；不存在 reset/rebase/整仓回滚入口。
- [ ] 仓库达到 80% 上限显示警告，达到 5 GiB 后停止新提交且不删除可达历史。
- [ ] Git 停用保留 `.git` 和历史；再次启用展示差异并创建 re-enable checkpoint。

## 5. 脚本与参数

- [ ] 支持的扩展名无需注册即可在原位置执行，工作目录为脚本父目录。
- [ ] Linux 和 Windows 默认执行器链按 PRD 顺序解析，可通过 YAML 指定绝对路径。
- [ ] 执行器不可用时回退；脚本一旦启动，非零退出不回退重跑。
- [ ] Windows Terminal 不参与执行器链；托盘入口不会影响服务日志捕获。
- [ ] shellwords 能正确处理空格、单双引号和反斜杠，不执行管道、重定向或命令替换。
- [ ] `{{NAME}}` 仅独占参数时解析；不存在变量、嵌入引用或非法名称被拒绝。
- [ ] 参数 16 KiB、变量值 4 KiB、256 参数、1,000 变量上限生效。
- [ ] Run 同时保存参数模板与当时实际参数数组。
- [ ] 脚本继承服务身份和环境，历史记录实际身份、执行器、路径和 SHA-256。

## 6. 并发、停止与状态

- [ ] 不存在队列或全局并发上限；不同脚本可立即并发。
- [ ] 人工重复启动同一文件 ID 时二次确认；确认后可并发。
- [ ] 执行状态只按冻结状态机迁移，终态不可逆。
- [ ] 第一次停止进入 stopping 并请求优雅退出；第二次停止强杀完整进程树。
- [ ] 默认无超时；指定超时后进入 timing_out，30 秒宽限后自动强杀。
- [ ] Linux 子进程组与 Windows Job Object 的子孙进程均被停止。
- [ ] 正常停服停止新工作、取消全部 Run，并在 30 秒后强杀残留。
- [ ] 异常服务退出尽量连带终止进程树；重启后旧活动 Run 变为 disconnected。
- [ ] Run 元数据没有网页删除入口，也不存在批量清空接口。

## 7. 日志与 SSE

- [ ] stdout/stderr 进入同一有序事件流，并保留来源标记。
- [ ] 页面刷新后使用 Last-Event-ID 继续，既不重放全部日志也不静默丢失可用事件。
- [ ] 不解析 ANSI、不提供 stdin 或 PTY；控制字符安全转义。
- [ ] 无效 UTF-8 不破坏页面，并显示编码错误标记。
- [ ] 达到 100 MiB 后保留 5 MiB 头部和约 95 MiB 尾部，显示截断及丢弃量。
- [ ] 日志达到上限后进程不会因 stdout/stderr 管道阻塞。
- [ ] 页面初始与 DOM 事件上限生效，暂停显示不暂停后台日志读取。
- [ ] 90 天或全局 1 GiB 触发日志清理时，Run 元数据保留并标记 log_expired。
- [ ] 日志写入失败时 Run 标记 log_incomplete，脚本不会被自动终止。

## 8. 快捷执行与计划

- [ ] 快捷项只能由管理员从历史 Run 主动保存，可命名、排序和删除。
- [ ] 正常快捷项单击即 POST 启动；同脚本运行时显示确认；失效路径不能启动。
- [ ] 五段 cron 校验、常用预设和未来五次预览使用实例统一时区。
- [ ] 服务停机期间不补执行，重启后从下一未来时间继续并记录错过汇总。
- [ ] 默认允许计划重叠；禁止重叠时只记录 skipped 触发，不创建 Run。
- [ ] 计划触发使用与手动执行相同的变量、执行器、Git 安全门、日志和状态模型。
- [ ] 未创建 Run 的触发明细一年后聚合，管理员不能删除。
- [ ] 不读取或修改系统 crontab。

## 9. 审计与历史

- [ ] 认证、文件、Run、计划、变量、快捷项、Git 和自动清理均产生审计事件。
- [ ] 审计事件包含时间、动作、目标、结果与来源地址，但不含敏感值或文件内容。
- [ ] 审计页面不能修改单条事件，默认保留一年并可下载。
- [ ] Run 历史永久不可删除；计划和快捷来源删除后仍有名称快照可解释。

## 10. 服务、托盘与配置

- [ ] Windows 服务默认 LocalSystem、自动启动；Linux systemd 服务默认 root、自动启动。
- [ ] 手动运行时继承当前用户且不因不是最高权限而拒绝启动。
- [ ] 服务安装、卸载、启停、状态、admin reset、config validate、doctor、version 命令可用。
- [ ] 卸载服务不删除配置、文件、数据库或 Git 历史。
- [ ] 不存在用户 backup/restore 或自动更新命令。
- [ ] Windows 托盘无主窗口、单实例，菜单与 PRD 一致；退出托盘不停止服务。
- [ ] 托盘区分服务进程运行与 HTTP 就绪；端口错误时显示异常并可打开日志。
- [ ] 默认配置优先级正确，命令行覆盖不写回 YAML。
- [ ] 同一状态目录第二个服务实例被排他锁拒绝。

## 11. SQLite、磁盘与升级

- [ ] SQLite 运行时 WAL、FULL synchronous、foreign keys 与 busy timeout 生效。
- [ ] Run 输出不逐事件写入 SQLite。
- [ ] schema 迁移前创建内部快照，迁移失败回滚并拒绝启动。
- [ ] 新 schema 不能被旧二进制静默降级。
- [ ] 状态卷低于 100 MiB 拒绝新执行；受管卷低于 100 MiB 拒绝上传/编辑。
- [ ] `doctor` 只读检查配置、目录、SQLite、日志、执行器、Git、端口、TLS、服务和磁盘，不泄漏敏感值。

## 12. 发布与兼容

- [ ] Windows amd64/arm64 ZIP 含服务与托盘二进制；Linux amd64/arm64 tar.gz 含服务二进制。
- [ ] 发布 SHA-256 文件；不依赖 Node.js、Redis、消息队列或外部运行时。
- [ ] Windows 10/11、Server 2019+ 与至少两个代表性 systemd Linux 发行版通过服务生命周期测试。
- [ ] 当前及前一个主要版本的 Chrome、Edge、Firefox、Safari 完成核心流程测试。
- [ ] 移动端可登录、浏览、启动、看日志、快捷执行和停止。
- [ ] 所有用户界面、错误、托盘和 CLI 帮助为简体中文，结构化标识保持英文。

## 13. 明确不验收

MVP 不验收多用户/RBAC、沙箱、公共 API、DAG、多服务器、Docker 正式部署、系统 crontab、Git 远程/LFS、用户备份命令、通知、插件、交互终端、目录打包、通用权限编辑、自动更新、多语言或正式多实例。
